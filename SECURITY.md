# Security Policy

## Supported Version

Security fixes are applied to the latest version on the default branch.

## Reporting A Vulnerability

Use GitHub's **Security** tab to submit a private vulnerability report. Do not
open a public issue containing exploit details, credentials, private save data,
player identifiers, or server information.

Include:

- the affected version or commit;
- clear reproduction steps;
- expected and actual behavior;
- the potential impact;
- any suggested mitigation.

## Read-Only Safety

Any path that can alter, replace, patch, serialize, or overwrite a Palworld save
is considered a security and product-boundary violation. Reports involving save
mutation will be treated with priority.
